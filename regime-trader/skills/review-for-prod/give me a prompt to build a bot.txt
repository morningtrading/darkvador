Build a modular crypto trading bot focused on Hyperliquid that can support multiple strategies as plug-ins, where strategies can be added, removed, or upgraded later without changing the core engine.

Goal
Create a production-minded MVP trading system that:

Connects to Hyperliquid for market data and trading.
Scans markets to identify the most promising crypto opportunities across multiple timeframes.
Supports multiple trading strategies through a clean strategy interface.
Allows new strategies to be added later as separate modules.
Includes strong risk management, paper trading, backtesting, and live execution support.
Produces clear analytics, rankings, and execution logs.
Core Requirements
1. Architecture
Design the bot with a modular architecture containing these components:

Exchange Connector
Connect to Hyperliquid REST/WebSocket APIs.
Fetch live prices, candles, order book, funding, open interest if available, and account/position data.
Support both testnet/paper mode and live mode.
Market Scanner

Rank coins by opportunity score across different timeframes such as:
5m
15m
1h

Strategy Engine

Each strategy should expose methods like:

generate_signal(market_data)
size_position(account_state, risk_rules)
should_exit(position, market_data)
on_fill(order_event)
Strategies must remain independent from the execution engine.
Execution Engine
Place, amend, cancel, and monitor orders.
Support market, limit, stop-loss, take-profit, and trailing logic where possible.
Handle slippage checks, duplicate order prevention, and retry rules.
Risk Manager
Enforce:
max risk per trade
max daily loss
max concurrent positions
max leverage
max exposure per coin
global kill switch
Block trading when volatility, spread, or liquidity conditions are unsafe.
Portfolio and Analytics Module
Track realized and unrealized PnL.
Compute win rate, profit factor, Sharpe ratio, drawdown, expectancy, and strategy-level performance.
Compare performance by strategy and timeframe.
Backtester
Run strategies on historical candle data.
Output metrics, trade logs, drawdown, and equity curve.
Make it easy to test newly added strategies.
Config System
Use environment variables plus config files.
Keep secrets secure.
Allow enabling/disabling strategies without changing code.
