Ajoute exactement ces deux features, dans cet ordre :

FEATURE 1 — vwap_distance
  Définition :
    vwap = cumsum(close * volume) / cumsum(volume)
    calculé sur une fenêtre roulante de 20 barres
    (pas un VWAP ancré sur la journée — on n'a pas
    garantie d'avoir l'open de session dans toutes
    les données)

    vwap_distance = (close - vwap) / atr_14
    où atr_14 = moyenne mobile 14 barres de (high - low)

  Propriétés attendues :
    - Causal : fenêtre roulante uniquement, pas de lookahead
    - Unité : multiple d'ATR (dimensionless)
    - Valeurs typiques : entre -3.0 et +3.0
    - NaN : les 20 premières barres (warmup roulant)

FEATURE 2 — volume_clock
  Définition :
    Pour US equities (session 09:30 → 16:00 = 390 minutes) :
      minutes_since_open = (bar_time.hour * 60
                           + bar_time.minute) - 570
      volume_clock = minutes_since_open / 390
      clippé entre 0.0 et 1.0

    570 = 9h30 en minutes depuis minuit

    Si le DatetimeIndex est timezone-aware (UTC ou autre),
    convertir en America/New_York avant le calcul.

    Si les données ne sont pas sur une session US
    (forex, crypto 24/7), mettre volume_clock = 0.5
    pour toutes les barres (neutre, pas de distorsion).

  Propriétés attendues :
    - Aucun NaN (valeur définie pour chaque barre)
    - Valeurs : [0.0, 1.0]
    - 0.0 = ouverture session, 1.0 = clôture session
