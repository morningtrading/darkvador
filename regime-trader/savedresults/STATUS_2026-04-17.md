## What was built this session

Three improvements were implemented and tested based on the diagnostic report:

### 1. Indices blend_exclude (config + backtester + live path)
Added `hmm.blend_exclude: [GLD, TLT, USO]` to `settings.yaml`.  
All blend sites (backtester `run()`, `run_grid()`, and all 3 live-path sites in `main.py`) now filter 
non-equity symbols before blending `log_ret_1` / `realized_vol_20`.  
**Result: No effect on Indices.** Regime distribution unchanged (still 8 labels, identical counts).  
Root cause: the blending itself changed the Indices HMM to see a differentiated basket signal — 
excluding GLD/TLT/USO from blend doesn't restore the SPY-only regime learned before blending.  
**Indices performance remains broken: +20.55%, Sharpe 0.070** (was +53% before blending was added).

### 2. SMA gate extended to NEUTRAL regime (regime_strategies.py)
Added second branch to gate:  
- `HighVolDefensive + above SMA-200` → MidVolCautious (unchanged)  
- `MidVolCautious + above SMA-200` → **LowVolBull** (1.25× leverage, new)  
**Result: Harmful for Stocks.**  
Without stops: +95.29% vs +108.17% pre-change (−12.88pp), MaxDD worsened −31.54% → −33.83%.  
The LowVolBull strategy adds 1.25× leverage during NEUTRAL, amplifying drawdowns when 
NEUTRAL transitions to CRASH/BEAR — precisely the worst moment to be leveraged.

### 3. Regime-conditional stops (backtester _run_single_window + _run_oos_sim)  
Added `_STOP_EXEMPT_REGIMES = {NEUTRAL, BULL, WEAK_BULL, STRONG_BULL, EUPHORIA}`.  
Stops now only fire in CRASH / BEAR / STRONG_BEAR / WEAK_BEAR.  
**Result: Interacted with change #2 — cannot isolate cleanly yet.**  
With stops + all changes: +113.62%, Sharpe 0.807, MaxDD −26.20%.  
vs old enforce_stops=True baseline: +106.40%, Sharpe 0.906, MaxDD −12.52%.  
Return improved but Sharpe and MaxDD both got worse — the gate adding leverage in NEUTRAL  
is building a larger position just before stops would have fired.
